> **버전**: v0.2 (상세화)
> **최초 작성**: 2026-03-13
> **현재 상태**: 데모 버전 (읽기 전용)

---

## 1. 개요

### 1.1 프로젝트 목적

매일 출근 후 오늘 해야 할 일과 앞으로 해야 할 일을 **한 화면**에서 빠르게 파악할 수 있는 업무 대시보드를 제공한다.
GitLab 커밋 이력과 Redmine 일감 데이터를 각 서비스 API로 가져와 통합 UI로 제공하며, AI 챗봇을 통해 업무 데이터를 기반으로 한 질의응답을 지원한다.

### 1.2 범위 (데모 v1)

- **읽기 전용**: 데이터 생성/수정/삭제 기능 없음
- **외부 API 연동**: GitLab API, Redmine REST API, Claude API (Anthropic)
- **단일 사용자**: 로그인 없이 브라우저 로컬 스토리지에 API 토큰 저장
- **SPA (Single Page Application)**: React 기반 프론트엔드만 존재 (별도 백엔드 서버 없음)

### 1.3 향후 확장 고려 (v2 이후, 현재 미구현)

- 다중 사용자 지원 (로그인/인증)
- 알림 푸시 (Slack, 이메일 웹훅)
- 일감 상태 변경 (쓰기 기능)
- 커스텀 위젯 배치

## 2. 사용자

### 2.1 주요 사용자

- 개발팀 구성원 (개발자, 팀 리더)
- 매일 GitLab 커밋 현황 및 Redmine 일감을 확인해야 하는 인원

### 2.2 사용 시나리오

1. 출근 후 대시보드를 열어 **오늘 내가 담당한 Redmine 일감** 상태를 확인한다.
2. 팀원들의 **최근 1주일 GitLab 커밋/머지 이력**을 조회하여 진행 상황을 파악한다.
3. 챗봇에게 "이번 주 완료된 일감 목록 알려줘" 처럼 자연어로 질의한다.
4. 설정 화면에서 API 토큰을 갱신한다.

## 4. 기능 요구사항

### 4.1 대시보드 홈 (신규 제안)

**목적**: 전체 업무 현황을 한눈에 파악

#### 요약 카드

- **오늘의 일감**: Redmine에서 담당자 = 나, 기간 내 포함인 일감 수
- **이번 주 커밋**: 나의 GitLab 커밋 수 (7일 이내)
- **진행 중 일감**: 상태 = "진행 중"인 Redmine 일감 수
- **미완료 일감**: 마감일이 오늘 이전이고 완료되지 않은 일감 수 (주의 표시)

#### 최근 활동 피드

- GitLab 이벤트(push, merge)와 Redmine 일감 업데이트를 시간순으로 혼합 표시
- 각 항목에 출처 아이콘(GitLab 로고 / Redmine 로고) 표시

---

### 4.2 GitLab 커밋 이력

**대상 URL**: `https://gitlab.ubware.com/`
**인증**: Personal Access Token (read_api 권한)

#### 4.2.1 프로젝트 선택

- 접근 가능한 전체 GitLab 프로젝트 목록 드롭다운 표시
- 선택된 프로젝트의 브랜치 목록 하위 드롭다운으로 연동
- 마지막 선택 값은 로컬 스토리지에 저장하여 재방문 시 유지

#### 4.2.2 커밋/이벤트 목록

- 선택한 프로젝트 + 브랜치의 최근 **1주일 이내** push / merge 이벤트 목록 표시
- 표시 항목:
  - 커밋 해시 (앞 7자리, 클릭 시 GitLab 링크로 외부 이동)
  - 커밋 메시지
  - 작성자 이름 + 아바타
  - 커밋 시각 (상대 시간, 예: "2시간 전")
  - 변경 파일 수

#### 4.2.3 필터링

- **사용자 필터**: 커밋 작성자 멀티셀렉트 필터 (체크박스 목록)
- **기간 필터**: 오늘 / 3일 / 7일 / 직접 입력 (날짜 범위 피커)
- **키워드 검색**: 커밋 메시지에서 키워드 검색

#### 4.2.4 추가 제안 기능

- **통계 차트**: 사용자별 커밋 수 막대 차트 (주간)
- **브랜치 비교 뷰**: 두 브랜치 간 커밋 차이 표시
- **즐겨찾기 프로젝트**: 자주 보는 프로젝트를 상단에 고정

---

### 4.3 Redmine 일감 리스트

**대상 URL**: `https://redmine.ubware.com/`
**인증**: API Access Key (사용자 계정별 발급)

#### 4.3.1 프로젝트 선택

- 접근 가능한 전체 Redmine 프로젝트 목록 드롭다운 표시
- 선택 후 해당 프로젝트의 **목표 버전(Target Version)** 드롭다운 연동
- 마지막 선택 값 로컬 스토리지 유지

#### 4.3.2 일감 트리 뷰

- 상위 일감 → 하위 일감 **트리 구조**로 표시
- 각 노드 표시 항목:
  - 일감 번호 (클릭 시 Redmine 링크로 외부 이동)
  - 제목
  - 담당자 아바타 + 이름
  - 상태 배지 (색상 구분: 신규=회색, 진행 중=파랑, 완료=초록, 반려=빨강)
  - 진행률 게이지 바
  - 마감일 (오늘 이전이면 빨간색 강조)
- 하위 일감이 있는 노드: 펼치기/접기 토글 아이콘

#### 4.3.3 필터링

- **담당자 필터**: 멀티셀렉트 (체크박스)
- **상태 필터**: 신규, 진행 중, 완료, 반려 다중 선택
- **우선순위 필터**: 낮음, 보통, 높음, 긴급
- **기간 필터**: 마감일 기준 (이번 주 / 이번 달 / 직접 입력)
- **키워드 검색**: 제목 내 키워드 검색

#### 4.3.4 추가 제안 기능

- **일감 요약 통계**: 상태별 일감 수 도넛 차트
- **마감 임박 일감**: 3일 이내 마감 일감 상단 경고 배너
- **완료율 표시**: 목표 버전 기준 전체 완료율 프로그레스 바
- **일감 내보내기**: 현재 필터 결과를 CSV로 다운로드

### 4.4 AI 챗봇 서비스

**모델**: Anthropic Claude (claude-haiku-4-5 또는 claude-sonnet-4-6)
**인증**: Anthropic API Key

#### 4.4.1 UI/UX

- 화면 우측 하단 **플로팅 버튼**으로 접근
- 클릭 시 채팅 패널 펼쳐짐 (화면 높이의 50% 차지)
- 다시 클릭 또는 X 버튼으로 접기
- 채팅 입력창: 하단 고정, Enter 키 or 전송 버튼으로 전송
- 메시지 스크롤: 최신 메시지가 항상 화면에 표시

#### 4.4.2 컨텍스트 주입

챗봇은 현재 로드된 업무 데이터를 컨텍스트로 받아 응답한다:

- 현재 선택된 Redmine 프로젝트의 일감 목록 (요약)
- 현재 선택된 GitLab 프로젝트의 최근 커밋 목록 (요약)
- 현재 날짜/시간

#### 4.4.3 지원 질의 예시

- "이번 주 완료된 일감 목록 알려줘"
- "담당자가 홍길동인 진행 중 일감은 뭐야?"
- "마감일이 지난 일감이 있어?"
- "어제 커밋한 사람은 누구야?"
- "가장 많이 커밋한 사람이 누구야?"

#### 4.4.4 추가 제안 기능

- **대화 기록 저장**: 세션별 대화 내역 로컬 스토리지 저장 (최대 50개 메시지)
- **대화 초기화 버튼**: 새 대화 시작
- **추천 질문 칩**: 챗봇 패널 상단에 자주 쓰는 질문 버튼 제공
- **마크다운 렌더링**: 응답 내 마크다운 형식(표, 목록) 렌더링

### 4.5 설정 (신규 제안)

**목적**: API 토큰 및 사용자 환경 설정 관리

#### 4.5.1 API 연결 설정

| 항목 | 설명 |
|------|------|
| GitLab URL | 인스턴스 URL (기본값: https://gitlab.ubware.com) |
| GitLab Personal Access Token | read_api 권한 필요 |
| Redmine URL | 인스턴스 URL (기본값: https://redmine.ubware.com) |
| Redmine API Key | 사용자 계정 > API 액세스 키 |
| Anthropic API Key | claude 모델 사용 |

- 입력값은 브라우저 로컬 스토리지에 암호화 없이 저장 (데모 버전)
- 연결 테스트 버튼: 입력된 토큰으로 API 호출 후 성공/실패 표시

#### 4.5.2 화면 설정

- **테마**: 라이트 / 다크 / 시스템 따르기
- **자동 새로고침 간격**: 수동 / 5분 / 15분 / 30분
- **언어**: 한국어 (향후 다국어 확장 가능)

### 9.2 핵심 차별화 포인트

#### 사내망 보안 우선 설계
GitLab/Redmine이 사내망에 있는 경우, 외부 SaaS 서비스는 데이터에 접근할 수 없다.
본 대시보드는 브라우저에서 Nginx 프록시를 통해 사내망 API를 직접 호출하므로 업무 데이터가 외부로 전송되지 않는다.

#### 기존 인프라 활용 (Zero Migration)
Jira/Linear로 마이그레이션 없이 현재 사용 중인 GitLab + Redmine 환경 그대로 연동한다.
설치/설정 시간 최소화 — 환경변수 3개(URL, Token, API Key)만 입력하면 즉시 사용 가능.

#### AI 챗봇 with 업무 컨텍스트
로드된 실제 업무 데이터(일감 목록, 커밋 이력)를 Claude에게 컨텍스트로 제공한다.
"이번 주 마감인 내 일감이 뭐야?" 같은 자연어 질의로 필터링/검색을 대체한다.

#### 데모 모드 즉시 체험
`VITE_DEMO_MODE=true` 한 줄로 API 키 없이 전체 기능을 체험할 수 있다.
도입 전 팀원 설득, 발표, 데모에 활용 가능.

### 9.3 목표 사용자 포지셔닝

```
[사내망 GitLab + Redmine 사용 중인 개발팀]
    ↓
현재: 매일 GitLab 탭 + Redmine 탭을 별도로 열어 현황 파악
문제: 두 서비스 간 컨텍스트 스위칭, 통합 뷰 없음, AI 활용 불가
    ↓
본 대시보드: 한 화면에서 커밋 + 일감 + AI 질의 통합
```
